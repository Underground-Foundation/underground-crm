# Security Policy

We take the security of Underground CRM and the political movements that rely on it very seriously. If you believe you have found a security vulnerability, please report it to us responsibly so that we can address it before it is publicly disclosed.

---

## Supported Versions

Only the latest minor release of the master/main branch is supported with security updates. We recommend all installations keep up-to-date with the latest version of the package.

| Version | Supported |
| --- | --- |
| 0.1.x | :white_check_mark: Yes |
| < 0.1.0 | :x: No |

---

## Reporting a Vulnerability

Please do not report security vulnerabilities via public GitHub issues. Instead, report them privately via email.

### Where to Report
Send vulnerability reports to **security@example.com** *(please replace with your official security contact address)*.

If possible, encrypt your email using GPG/PGP. *(You can add your GPG key details or fingerprint here).*

### What to Include
To help us understand and resolve the issue quickly, please include the following information in your report:
* A detailed description of the vulnerability.
* The impact and potential exploit scenarios.
* Step-by-step instructions (or a proof-of-concept script/exploit) to reproduce the vulnerability.
* Any potential mitigations or fixes you suggest.

---

## Our Security Process

1. **Acknowledgment**: We will acknowledge receipt of your report within 48 hours and coordinate a plan for investigation.
2. **Evaluation**: Our maintainers will validate and evaluate the vulnerability.
3. **Fix and Release**: Once verified, we will work on a fix. A security release will be issued to address the issue.
4. **Coordinated Disclosure**: We aim to resolve vulnerabilities within 30 days of receipt, after which a coordinated disclosure can be published. We ask that you do not share details of the vulnerability publicly until a patch has been released.
